# AGENTS.md

本文件为本仓库内 AI 编码代理与协作者的工作指南。适用于整个项目，除非子目录中另有更近的 `AGENTS.md` 覆盖。

## 项目定位

本项目是一个网页端八字测算工具。v1.0 目标是让用户通过网页输入出生信息或完整四柱八字，系统完成排盘、AI 解读，并生成可下载 PDF 报告。

当前主要需求来源是：

- `软件开发说明书.md`
- `软件开发说明书.docx`
- `assets/software-spec/` 中的界面参考图

在需求冲突时，优先遵循最新的用户明确指令；没有新指令时，以 `软件开发说明书.md` 为主要依据。

## 推荐技术方向

说明书推荐的默认技术栈如下。新建工程骨架时应优先沿用，除非用户明确要求调整。

- 前端：Next.js + React + TypeScript
- 样式：Tailwind CSS 或 CSS Modules
- 表单：React Hook Form + Zod
- 后端：Python FastAPI
- 数据库：优先 PostgreSQL；v1.0 快速交付可用 SQLite，但模型应保持可迁移
- 后端模块建议：`services/chart`、`services/llm`、`services/report`
- LLM：使用 OpenAI-compatible API 抽象，通过环境变量配置
- PDF：由后端生成正式报告，前端只触发生成与下载流程

如果未来仓库中出现实际脚手架、包管理器或测试框架，优先遵循现有工程配置，而不是重新引入一套并行方案。

## 关键业务原则

- 排盘必须由确定性算法或可靠历法库完成。
- 大模型只负责基于后端排盘结果生成命理解读、综合建议和报告文案，不得自行重算四柱。
- 公历、农历、四柱手动输入三种模式都需要清晰建模。
- 立春交界、节气交界、夜子时、闰月、未知时辰是真实核心场景，不能作为边缘需求忽略。
- 未知时辰时，不得强行推断时柱；依赖时柱的分析应降级或标明不确定。
- 真太阳时用于时辰判断，页面需展示校正后的真太阳时和出生地经纬度。
- 命理分析仅作传统文化参考，所有 AI 解读和 PDF 报告必须包含免责声明。

## 安全与合规

- 不得在前端暴露 `LLM_API_KEY` 或任何服务端密钥。
- 后端日志不得记录完整 API Key、敏感 token 或过量用户输入。
- 所有用户输入都要做长度限制、格式校验和基础清洗。
- PDF 下载地址应有有效期或访问控制策略。
- 涉及健康、财务、感情、法律等内容时，表达必须中性，避免极端、绝对化或替用户做专业决策。
- 不要提交真实密钥、个人隐私数据、生成的临时报告文件或本地环境文件。

## API 与数据约定

后端接口应优先采用说明书中的统一响应格式：

```json
{
  "success": true,
  "data": {},
  "requestId": "req_20260703213000001"
}
```

失败响应：

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "出生时间不能为空",
    "details": {}
  },
  "requestId": "req_20260703213000001"
}
```

核心接口优先保持如下语义：

- `GET /api/geo/search`：地区检索，返回经纬度和时区。
- `POST /api/chart/calculate`：根据出生信息或手动四柱生成标准命盘。
- `POST /api/analysis/generate`：基于命盘生成 AI 解读。
- `POST /api/report/pdf`：生成 PDF 报告。

核心数据结构应覆盖：输入资料、四柱、十神、藏干、纳音、神煞、大运、流年、流月、AI 解读和 PDF 报告状态。

## 前端实现指南

- 第一屏应直接是可用的排盘表单，不做营销式落地页。
- 移动端和桌面端都必须可用，复杂表格在移动端允许横向滚动。
- 首页字段至少覆盖：姓名、性别、输入模式、出生时间、出生地区、真太阳时、经纬度、开始排盘。
- 结果页必须清晰展示年柱、月柱、日柱、时柱，并明确标识日柱为日主。
- 大运、流年、流月应有当前项高亮。
- LLM 失败时，用户仍应能查看排盘结果。
- PDF 下载按钮文案使用“下载 PDF 报告”。

## 后端与算法指南

- 排盘计算逻辑与 LLM 调用逻辑要分离。
- 历法和节气计算优先使用可靠库或可验证数据源；不要用大模型或硬编码猜测结果。
- 年柱以立春为年份分界，不以农历正月初一为分界。
- 月柱以节气为月份分界。
- 夜子时规则、真太阳时校正和起运顺逆必须有单元测试覆盖。
- 地区解析失败、历法转换失败、LLM 超时、PDF 渲染失败都应返回明确错误码。

## 测试要求

- 首页表单校验

- 农历排盘
- 闰月
- 立春交界
- 节气交界
- 夜子时

- LLM 超时或失败降级

## 验收基线

v1.0 至少应满足：

- 用户能在网页端完成一次排盘。
- 系统能展示四柱、大运、流年、流月。
- 系统能生成 AI 解读。
- 系统能生成并下载 PDF 报告。
- 手机端和电脑端均可正常使用。
- 所有核心错误场景有明确提示。
- AI 解读和 PDF 报告包含免责声明。
